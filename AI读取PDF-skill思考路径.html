A reader's

PDF Skill

A field guide to the four-layer cognitive path an artificial reader walks—from a user's plain request, through indices and routing rules, down into the scripts that do the actual work.

EDITOR Jettlin SCRIBE Claude Opus 4.7 PRESS Skill 拆解

当用户说"帮我处理这个 PDF"，AI 没有立刻动手。它走的是一条四层的思考路径——先匹配，再读手册，再分流，最后才下沉到具体的工具与脚本。这条路径看上去像在读一本图书馆里的工具手册：先翻封面，再翻目录，再翻到对应的章节，最后跟着章节指引下到附录里的脚本。

这份文档把这条路径拆成四章，每一章对应一层判断。读完之后，你不仅能解释 AI 怎么用一个 skill，更能反过来推出：怎样写一个 AI 用得顺的 skill。

"A skill is not a body of knowledge. It is a route—and the reader must be able to walk it without getting lost." — Field Notes, Folio I

PDF skill 的 description 是怎么写的

这是 PDF skill 实际的 description（节选）。注意它不是抽象介绍，而是把所有可能触发它的具体动词都列了一遍：

Use this skill whenever the user wants to do anything
with PDF files. This includes reading or extracting
text/tables from PDFs, combining or merging multiple
PDFs into one, splitting PDFs apart, rotating
pages, adding watermarks, creating new PDFs,
filling PDF forms, encrypting/decrypting PDFs,
extracting images, and OCR on scanned PDFs...
If the user mentions a .pdf file or asks to produce one,
use this skill.
Lesson · 启示 description 不能写成"这是一个 PDF 工具"。AI 用关键词匹配，所以你必须把所有可能触发它的具体动词都列出来——读、合并、拆分、旋转、水印、填表单、加密、OCR……动词越穷尽，命中率越高。

索引 3 是最关键的

SKILL.md 里有几句话是路由指令。AI 必须严格执行，不能略过：

SKILL.md 原文AI 的理解触发动作
"For advanced features... see REFERENCE.md" 目录指针，遇到高级任务才读 暂不读，记下来
"If you need to fill out a PDF form, read FORMS.md and follow its instructions" 强制指令，填表单必须先读 立即跳转 forms.md
Marginalia · 旁注 这就是为什么 SKILL.md 不必把所有细节写完。它只需做三件事：① 让 AI 知道有什么能力；② 给最常用的快速代码；③ 把复杂分支甩到子文档。AI 读完 SKILL.md 是"看完地图"，不是"看完百科全书"。

Lesson · 启示

为什么这样设计？

① 脚本能做的就别让 AI 做——抽坐标、转格式、检查重叠是机械活，AI 临时写代码容易在小数点和坐标系上出错；

② AI 能做的就别写死在脚本里——"这个空格旁边写着'出生日期'，所以应填生日"这种语义判断脚本做不了；

③ 两者交替出现，每一步的输出都是下一步的输入，AI 不会跑偏。

常见错误对照

常见错误卡在哪一层正确做法
description 只写"PDF utility" 第 I 层匹配不上 列出所有可能触发的动词
SKILL.md 写成 5000 行的百科 第 II 层读不准 核心章节 + 子文档跳转
章节用 "Section 1, Section 2" 第 III 层对不上号 用任务名做标题
所有逻辑都塞在 SKILL.md 第 IV 层临时写代码出错 机械活拆成脚本
子文档间没清晰跳转 第 II / IV 层不知何时读 用 "If X, read Y.md"
